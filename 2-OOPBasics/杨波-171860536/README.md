这次用到的类有World，CalabashBrother和最后运行main的CalabashBrothersSort

CalabashBrother代表葫芦娃，用有属性：
身份，id，唯一标志身份名和颜色，所以最后排序时可以根据id来排
接触范围，touchSize，表示在这个范围内能与其他物体接触
实际范围，realSize，表示葫芦娃实际占的位置
位置数组，position，表示葫芦娃中心所处位置，上述两个范围都是针对这个中心来的
速度，speedX和speedY，表示葫芦娃移动速度
还有个方向，这个在移动时会用到

World类用于构建一个世界，这里用16*16的方格来表示，每个方格变长为100，地形都设为平地
然后在世界中创建7个葫芦娃，并有一个数组calabashList按位置顺序记录各个葫芦娃

最后运行的CalabashBrothersSort类，执行排序事件

讲一下主要运行过程，在CalabashBrothersSort类中执行，按照排序判断，id为i(或j)的的葫芦娃是否需要走在某个位置，
如果要走的话就执行葫芦娃中的walkTo方法走到对应位置，直至排序完成，所以主要的内容还是在walkTo，下面简单介绍一下。
首先根据行走方向判断对应方向上是否与别的葫芦娃接触（用到接触范围，方向等变量），有的话与其交换位置，否则直接走
交换位置时，首先让两个葫芦娃错开位置，以两个葫芦娃上下相邻为例，因为一个占100*100，正常情况下葫芦娃都位于格子中心，
所以一个葫芦娃往左走50，一个葫芦娃往右走50，然后分别向下向上走，最后再右走50，左走50回到中心，完成交换。
主要思路就是这样了。
